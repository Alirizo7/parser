# Подводные камни, бизнес-решения, история

## Расхождения с исходным ТЗ (подтверждены реальными файлами — верим файлам)

1. **Номер РМ и должность — из СОДЕРЖИМОГО документа** (абзац `000002-сонли …` /
   «ИШ ЖОЙИ НОМИ»), НЕ из имени файла. Имена ненадёжны (`3_Умумий_…doc`,
   `Energetik.doc` при `Montajchi` внутри). Папка карт = где больше всего
   кандидатов. Карта без строки `…-сонли` → громкое предупреждение, не тихая потеря.
2. **«а»-суффиксные РМ** (`000012а`) сортируются сразу после базового; при пустых
   факторах/отсутствии карты данные копируются из базового (с предупреждением).
3. **Код должности** — из документа «Перечень» (по номеру), не из карты.
4. **Льготы (колонки 21–25 в 6_5) — прямо из карты, раздел 4** (не эвристики):
   таблица 4.1 + текст 4.2 (пенсия). ⚠️ Молоко: эталон Бухоро вручную ставит «ҳа»
   всем медикам (20 РМ) вопреки форме карты — мы выводим значение карты, эти ячейки
   в самопроверке идут в notes.
5. **Травмоопасность c18** — единственная эвристика: «медик → 1, иначе → 2», всегда
   флаг `injury_risk_heuristic`. (Для 6_4 класс травмоопасности читается из п.2.3
   карты напрямую — это ДРУГОЕ поле, `injury_risk_class_6_4`.)
6. **Должность в 6_5** — без ручных переносов-дефисов эталона (`ҳисоб-лаш`);
   самопроверка сверяет без дефисов/пробелов.
7. **Должность и код в 6_5 — из «Перечня»**; расхождение с картой → флаг
   `position_mismatch`, код молча не подменяется.
8. **Реквизиты компании всегда перезаписываются; при неудаче ячейка очищается** —
   СТИР/ИФУТ/МХБТ компании-образца из шаблона не «протекают» другому клиенту.

## Латинский клиент (RIZOYEV) — расхождения его эталона с его же картами

Эталоны RIZOYEV — ручные документы; местами противоречат исходным картам. Наш вывод
верен картам; такие поля идут в notes самопроверки и подсвечиваются в ревью:

- **5_1б, РМ 000001/000005/000007** — в картах нет раздела веществ (в эталон клиент
  вписал вручную). РМ выводится с пустой ячейкой вещества + флаг `substances_missing`.
- **Аэрозоли/вибрация/ЭМ-поле** — эталон местами противоречит картам (пыль 1.1.6 не
  отнесена к аэрозолям; локальная вибрация 1.3.5 записана как общая). Извлекаем по картам.
- **Названия веществ** клиент сокращает (`Silikat changi (loy)` → `Silikat changi`);
  единицы (`mg/m3`) срезаются, скобочные уточнения мы сохраняем.

## Документ 6_4 — ключевые решения

- Таблица строится **полностью из данных батча**: шаблон-ассет несёт только шапку и
  прототипы блоков; список подразделений собирается заново группировкой позиций
  «Перечня». Блоки чужого клиента из ассета удаляются, недостающие добавляются.
- Люди (ходимлар/аёллар) считаются **из колонок «Перечня»**, не из карты; карта даёт
  только 3 показателя распределения (класс условий, класс травмоопасности, статус ЯТҲВ).
- «а»-суффиксные строки весят 0 в счёте рабочих мест (та же единица, доп. смена).
- «Мос» (c9) и «Ятҳв кўзда тутилмаган» (c11) — независимые, пересекающиеся величины
  (эталон: 114 и 93). c12 намеренно пуст (как в эталоне).
- Строка «Перечня» без карты исключается из 6_4 с громким warning (НЕ синтезируется).

## Хрупкие места (сводка по всем модулям)

**Извлечение/нормализация:**
- Невидимые символы вшиты в код буквально: U+200E внутри regex `_extract_pension`
  (extract.py), **soft hyphen U+00AD** в regex сравнения должностей (selfcheck.py).
  При копировании/рефакторинге легко потерять.
- Семантика `-`: `is_empty` считает его пустым; в трёх местах специально обходится
  (реквизиты, колонка класса факторов, колонка «Аёллар»).
- `canon_yesno`: пусто/мусор → «йўқ»; префиксы `ха`/`да` могут ложно дать «ҳа» на
  длинном тексте.
- `to_cyrillic`: `yo` в конце строки → `йо` (не `ё`); типографские апострофы
  `’ ‘ ʼ ´ ʹ` не распознаются в `o'/g'`.
- Часть якорей матчится `.lower()`, а не fold (см. `03-EXTRACTION.md`, «Исключения»).
- `_CARD_NAME_HINT` (pipeline) обязан включать запятую в разделители — фикс потери
  РМ 000014/000055 (112 вместо 114).
- `_PERECHEN_WP_RE` обязан допускать латинский суффикс — фикс потери строк
  «000012a» в 6_4 (Sud-tibbiyot: 5 чел./2 жен. терялись).

**Рендер:**
- Прямые lxml-манипуляции (`_tr`, `_tbl`) — хрупко к смене python-docx.
- Жёсткая геометрия шаблонов: 5_1б шапка 2 строки; 6_5 прототипы R2/R3 (данные
  клонируются из строки НУМЕРАЦИИ — обход бага смещённого объединения «Сут»);
  6_4 итог R3–R6, блоки с R10 шагом 4. Пересвёрстка ассетов сломает индексы.
- Транслитерация вывода не трогает колонтитулы/textbox-ы.

**Веб/оркестрация:**
- Daemon-поток (thread-раннер) умирает при рестарте — батч зависает в PROCESSING;
  у Celery-задач нет retry/лимитов — упавший воркер тоже оставит PROCESSING.
- `Batch.error` — двойное назначение (ошибки + warnings); при перегенерации
  6_4-warnings могут дублироваться.
- `edit_cell`: read-modify-write всего JSON без блокировки; select не валидируется.
- `_set` через `queryset.update()` обходит `auto_now` — `updated_at` фоновыми
  задачами не двигается.
- Степпер прогресса матчится по подстрокам stage-строк без первой буквы.
- Фронт зависит от CDN (Tailwind play, unpkg, rsms.me).

**Excel-протоколы (5 xlsx, `render_excel_*` + `xlsx.py`):**
- Якорь разряда зрит. работ — `«иш оқими»`, НЕ `«визуал иш оқими»`: у Бухоро в
  картах опечатка `«Визувл иш оқими III»`, по полному якорю строка не находилась и
  разряд (кол. D файла 4) терялся у ВСЕХ РМ. Правьте якорь по устойчивой части.
- **Формулы псевдозамеров и текстовый факт**: `=H±шаг`/`=I±шаг` требуют, чтобы
  факт (H/I) был ЧИСЛОМ. При факте-тексте («йўқ») формулы нельзя оставлять —
  Excel даст `#VALUE!`. `_measure_fill`/цикл веществ файла 1 подменяют E/F/G/H
  тем же текстом (проверяйте оба места при правке).
- **Транслитерация в кириллицу искажает римские числа**: `to_cyrillic('IV')='ИВ'`,
  `'Ib'→'Иб'`. `xlsx.transliterate_region` при `cyr` пропускает ячейки по
  `_KEEP_LATIN_RE` (разряд/категория). `to_latin` безопасен (римские — латиница).
- Порядок в рендере: сначала ПИШЕМ значения ячеек, потом создаём merge — запись в
  merged-ячейку openpyxl запрещена (raise). `clear_body` снимает объединения с
  `max_row >= from_row` (иначе висящий спан после `delete_rows`).
- **Маска аномалий эталона — точечная** `(файл, РМ)` (`_EXCEL_ANOMALY`), НЕ по всему
  РМ: у 000051 в файлах 1/2/4 факт/время смещены (эталонная опечатка → notes), но
  в файлах 3/5 данные верны — их регресс ДОЛЖЕН провалить гейт.
- Файл 4 сознательно НЕ выводит пульсацию (1.11.6) и класс естеств. КЕО — эталон
  их не сообщает (0/52). Файл 5: формулы псевдозамеров есть только на прото-строке
  1.4.10; при заполненном 1.4.1–1.4.9 у другого клиента их колонки будут пусты
  (данные не теряются).
- Ассет держится на ЛАТИНИЦЕ эталона Суд (docx-ассеты — на кириллице эталона
  Бухоро). Оба через «транслитерацию к целевому письму» → работают на обоих клиентах.

**Инфраструктура:**
- LibreOffice ~1.8 ГБ RAM-спайк → concurrency=1, swap 2 ГБ, RAM-порог audit.sh.
- `/media/` раздаётся nginx без авторизации.
- Пароль Redis в `.env.prod` дублируется в 3 переменных.
- `DJANGO_SUPERUSER_PASSWORD` применяется только при первом создании.
- CI-деплой не вызывает deploy.sh; `.env.prod` не в `.dockerignore` (риск попадания
  в слои образа).
- Фикстуры selfcheck не в git — регрессия воспроизводима только локально.
