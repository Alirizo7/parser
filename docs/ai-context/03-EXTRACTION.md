# Извлечение: extract.py + справочник якорей mapping.py

## Общий приём

Каждый .docx раскладывается в чистую структуру `Doc` (dataclass):
`tables: list[list[list[str]]]` (таблица → строки → тексты ячеек) +
`paragraphs: list[str]`. Затем по декларативным якорям из `mapping.py`
(импортируется как `M`) достаются поля. Зависимости — только python-docx и stdlib.

Все якорные поиски двуязычны через `normalize.fold` (кириллица ≡ латиница), кроме
нескольких мест, где матчинг идёт по `.lower()` (см. «Исключения из fold» ниже).

Помощники поиска: `find_table(tables, anchor)` — первая таблица с якорем в любой
ячейке; `find_row(grid, anchor)`; `find_row_tokens(grid, tokens)` — строка, где
присутствуют ВСЕ токены; `row_value_after_label(row, label)` — первая непустая
ячейка, не содержащая метку (⚠️ `is_empty` трактует `-` как пусто); `last_value(row)`.

## Номер РМ и «а»-суффиксы

- Канон: `canonical_workplace_no(12, "а") == "000012а"`; разбор —
  `split_workplace_no("000012а") == (12, "а")`; сортировка — `workplace_sort_key`:
  `000011 < 000011а < 000011б < 000012` (нераспознанные — в конец).
- **Номер берётся ТОЛЬКО из содержимого** карты по regex `_WORKPLACE_CONTENT_RE`:
  `([0-9]{4,6})\s*([а-яёА-ЯЁa-zA-Z]?)\s*[-—–]?\s*(?:сонли|sonli)` — матч по СЫРОМУ
  тексту (не fold), чтобы сохранить кириллический суффикс «а». Нет матча →
  `workplace_no=""` → карта в «НЕ РАСПОЗНАНЫ» (громкое предупреждение pipeline).
- Имя файла (`parse_card_filename`: число+суффикс+разделитель+название) — только
  запасной источник суффикса и названия должности; номер из имени игнорируется.
  Причина: имена ненадёжны (`3_Умумий_…doc`, `Energetik.doc` при `Montajchi` внутри).

## Реквизиты компании

`extract_company_data`: таблица находится по якорю `корхонанинг номи`; для каждого
поля `COMPANY_FIELDS` — строка + `_reqs_value` (первая непустая ячейка ≠ метки;
**сохраняет `-`**). Коды: вся таблица склеивается в строку, ищутся
`(?:СТИР|STIR) / (?:ИФУТ|IFUT) / (?:МХБТ|MX[БB]T)` + хвост `\s*[:\-]?\s*([0-9][0-9 ]*)`
— цифры могут содержать пробелы, они удаляются.

## Таблица факторов (сердце карты)

- Поиск таблицы `_find_factor_table`: в первых 3 строках должен быть якорь `класси`
  И один из `("меҳнат шароитлари класси", "гигиеник меъёр")` (матч через `.lower()`!);
  фолбэк — самая длинная таблица документа.
- Колонки `_factor_columns`: дефолт name=1, actual=3, duration=4, класс = последняя;
  уточняются по строке-заголовку (якоря `ҳақиқий`, `давомийлиги`, `омиллари`, `класси`).
- `_parse_factor_rows` → `FactorRow(section, c0, name, actual, duration, cls)`;
  номер раздела из col0 «протягивается» вниз на под-строки без номера.
- `_factor_values` двухфазно:
  1. **Подытоги** `FACTOR_SUBTOTALS` — поиск строки по НАБОРУ токенов (не цельной
     фразе — грамматика кир/лат чуть различается: «Микроиқлим умумий» ↔
     «Mikroiqlimni umumiy»); значение берётся из КОЛОНКИ КЛАССА, а не «последней
     непустой» (иначе при пустом классе протёк бы текст объединённой ячейки-метки).
     Нет строки → `"-"`; значение через `normalize_class`.
  2. **Факторы по разделу** `FACTOR_SECTIONS` — `max_class` среди строк, чей раздел
     принадлежит префиксу. `section_matches`: `1.1.3 ∈ 1.1`, но `1.11 ∉ 1.1`.
- `_extract_substances` (для 5_1б): под-строки раздела `1.1.*` с непустыми name и
  actual; имя чистится от единиц (`clean_substance_name`), процент =
  `normalize_number(duration)`. Сортировка по `SUBSTANCE_ORDER` (углерод оксиди
  первым; lookup — точный lower, латинские имена уходят в «неизвестные», rank 100).
  Дополнительно к `name`/`pct` кладёт `norma`/`actual`/`cls` (для Excel-протокола 1).

## Построчные замеры для Excel-протоколов (файлы 2–5)

`FactorRow` несёт `norm` (колонка «Гигиеник меъёр», индекс 2 — `_factor_columns`
добавляет `norm_col`, класс-колонка по-прежнему `[3]`). `_measurement(fr)` →
`{norma,actual,time,cls}`. `_row_by_section` находит строку раздела (exact/prefix
номер, опц. с непустым фактом и якорем имени). Экстракторы (якоря — в
`mapping.*_SECTIONS`): `_extract_physical_measurements` (1.3.2 шум по имени
«шовқин товуш даражаси» — отличить от инфразвука под тем же 1.3.2),
`_extract_microclimate_measurements` (активная 1.8.x с непустым фактом + метка из
1.7.x того же номера; теплоизлучение 1.8.6 может быть текстом «йўқ»),
`_extract_lighting_measurements` (разряд «Vizual ish oqimi III..IX» —
`_LIGHTING_DISCHARGE_RE`), `_extract_em_measurements` (1.4.1..1.4.10). Все —
двуязычны через `fold`.

## Подразделение и численность

- `_extract_subdivision`: таблица `таркибий бўлинма` → строка `таркибий бўлинма
  номи` → значение → `canonical_subdivision` (алиасы `SUBDIVISION_ALIASES`,
  полное fold-равенство: «Рентген кабинет» → «Рентгент бўлими» — да, с лишней «т»,
  так в эталоне клиента).
- `_extract_employee_counts` → `(workers, female)`, оба `""` если не найдены (НЕ «0» —
  render_6_4 должен отличать «неизвестно» от нуля):
  - workers: строка `ишловчилар сони`;
  - female: строка `жумладан аёллар`; если её нет — вариант карты, где «Аёллар» —
    колонка с подписью СТРОКОЙ НИЖЕ строки workers: берётся `grid[idx+1]`, ищется
    колонка `аёллар`, значение из той же колонки строки workers. Специально НЕ
    `last_value`: `-` в колонке «Аёллар» = настоящий ноль женщин.

## СИЗ и льготы

- `_extract_ppe` → `"ҳа"/"йўқ"`: таблица `ятҳв`; колонка по якорю `мавжудлиги` —
  ⚠️ в шапке слово встречается ДВАЖДЫ (нужная колонка «Ходимда ЯТҲВ мавжудлиги» и
  «Сертификат … мавжудлиги») — берётся ПЕРВОЕ совпадение (нужная левее). «ҳа», если
  хоть в одной строке `canon_yesno(cell) == "ҳа"`.
- `_extract_benefits`: таблица `преференсиялар`; колонка факта — первая с якорем
  `мавжудлиги` («Ҳақиқатда/Ҳақиқий мавжудлиги» — у клиентов по-разному); строки по
  `BENEFITS_ROWS`; дефолт всех льгот — `"йўқ"`.
- `_extract_pension`: абзац с «пенсия таъминоти»/«пенсия таминоти», значение из
  пропуска между `ҳуқуқи|хуқуқи|huquqi` и открывающей скобкой; чистка regex
  `r"[_‎\s]+"` — ⚠️ внутри класса невидимый U+200E (LRM), не потерять при рефакторинге.

## Показатели специально для 6_4 (читаются напрямую из карты)

Не путать с эвристикой `injury_risk` и таблицей СИЗ (`ppe_provided`) — те для 5_1б/6_5.

- `_extract_injury_risk_class_6_4` → `"1"/"2"/"3"/""` из п.2.3. Опознание абзаца:
  начинается с `2.3` + любое из `INJURY_RISK_KEYWORDS` (у клиентов две формулировки);
  legacy-путь — оба якоря `INJURY_RISK_CLASS_ANCHORS` (когда номер пункта потерялся
  при конвертации). Значение: пропуск `____N____` (`_INJURY_CLASS_BLANK_RE`), фолбэк —
  цифра перед «дараж/daraja». **Намеренно без дефолта**: `""` → громкое предупреждение
  в render_6_4, а не тихое отнесение к произвольному классу.
- `_extract_ppe_status_6_4` → `'mos'/'mos_emas'/'kutilmagan'` из п.3.3. Ключевая
  семантика (история двух исправленных багов в docstring): п.3.3 — статичное
  УТВЕРЖДЕНИЕ «Иш жойи ЯТҲВ билан таъминланганлик талабларига жавоб беради» без
  переменной; сам факт наличия абзаца = «Мос» (на эталоне «Мос»=100% РМ). Явный
  вписанный ответ приоритетен: `yo'q` → `mos_emas`. Абзаца нет вовсе → `kutilmagan`.
  Поиск ответа: инлайн-хвост после якоря + следующие 2 абзаца до раздела IV
  (`кафолатлар`). Да/нет-токены: YES `{ha, bor, xa, da, yes}`, NO `{yoq, yuq, yok, no}`.
- `_extract_ppe_not_envisaged_6_4` → bool из ТАБЛИЦЫ СИЗ п.3.2: наименование СИЗ —
  колонка `row[1]`; категории («Мажбурий:», «Қўшимча:») и строки с маркером «Кўзда
  тутилмаган» пропускаются; ни одного реального СИЗ → True. НЕЗАВИСИМ от статуса
  «Мос/Мос эмас»: на эталоне «Мос»=114 и «кўзда»=93 пересекаются намеренно.

## Эвристики

- `is_medical(position, subdivision)`: подстроки `MEDICAL_KEYWORDS` в lower
  (⚠️ НЕ fold — латинские карты этой эвристикой не ловятся). Подразделение
  учитывается, т.к. группа «Врачлар, ўрта ва кичик тиббиёт ходимлари» относит к
  медицине и немедицинские должности внутри неё.
- `_injury_risk(medical)`: медик → `"1"`, иначе → `"2"`. Флаг ставится всегда.

## «Перечень»

- `_perechen_header_columns`: шапка склеивается ПО ВЕРТИКАЛИ из первых 4 строк на
  колонку, матч `.lower()`-подстрокой. Защиты: `position` не берётся из колонки, где
  есть `коди`; `female` — где есть `ишловчилар сони`. Дефолты: workplace=0,
  position=1, code=2, workers=3, female=4.
- `parse_perechen` → `{workplace_no: {job_code, position}}`; таблица — самая длинная;
  строка данных = ячейка workplace матчит `^\d{4,6}[а-яёА-ЯЁa-zA-Z]?$` — ⚠️ суффикс
  допустим И латиницей (фикс: латинские «а»-строки выпадали из 6_4 целиком, терялись
  люди). Дубли номера СХЛОПЫВАЮТСЯ (последний выигрывает).
- `parse_perechen_positions_6_4` → `(positions, warnings)` — позиции ПО ПОРЯДКУ
  документа: `{"workplace_no", "subdivision", "employees_count", "female_count"}`.
  Работает напрямую с docx XML: строка-разделитель подразделения = настоящее
  горизонтальное слияние на всю ширину (`len({id(c._tc) for c in cells}) == 1`) +
  текст не похож на номер. Дубли номеров СОХРАНЯЮТСЯ (намеренная асимметрия с
  `parse_perechen` — «две карты одной "а"-позиции на разные смены»).

### ⚠️ Мёртвые константы mapping.py (правка НЕ повлияет на извлечение)

- `COMPANY_CODES` — дубликат; реально коды извлекает `_CODE_LABELS` в extract.py.
- `WORKPLACE_NO_RE` — дубликат; реально работает `_WORKPLACE_CONTENT_RE` в extract.py.
- Хвостовой комментарий про «пенсия = ионизирующее излучение» описывает УСТАРЕВШУЮ
  эвристику; пенсия давно читается из раздела 4.2 карты.

### Исключения из fold (матчинг по `.lower()`, латиницу словят только если якорь совпал буквально)

`FACTOR_TABLE_HEADER_ANCHORS` (в `_find_factor_table`), `PERECHEN_COL_*`
(в `_perechen_header_columns`), `SUBSTANCE_ORDER` (точный lower-lookup),
`MEDICAL_KEYWORDS` (lower-подстрока). Утверждение «один набор якорей для двух
письменностей» строго верно только для fold-путей.
