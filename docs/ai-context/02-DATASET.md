# Схема датасета (единый промежуточный JSON)

Датасет = `company_data` (dict) + `workplaces` (list[dict], одна запись на РМ).
Создаётся `extract.extract_card()` + дополняется `pipeline.run_pipeline()`.
Хранится в `Batch.company_data` / `Batch.extracted_data`; выгружается командой
`extract_demo` как `{"company_data", "workplaces", "warnings"}`.

## company_data

| Ключ | Источник | Пример (реальный) |
|---|---|---|
| `name` | строка «Корхонанинг номи» таблицы реквизитов | `"Бухоро шаҳар тиббиёт бирлашмасига қарашли 1-сон оилавий поликлиника"` |
| `parent` | «Юқори турувчи корхона» | `"Ўзбекистон Республикаси Соғлиқни сақлаш вазирлиги"` |
| `address` | «Юридик манзили» | `"Бухоро вилояти, Бухоро ш., Арабон кўчаси, 2-уй"` |
| `product` | «Асосий маҳсулот тури» | `"Умумий шифокорлик амалиёти соҳасидаги фаолият"` |
| `stir` | regex по склеенной таблице; цифры, пробелы внутри удаляются (`311 452 380`→`311452380`) | `"204986878"` |
| `ifut` | то же | `"86210"` |
| `mxbt` | то же (метка бывает смешанной: `MX[БB]T`) | `"1706401"` |

Извлекается из **первой удачной карты** батча. Не найдено → пустая строка (при
рендере ячейка шаблона **очищается**, чтобы реквизиты компании-образца не «протекли»).

## Запись рабочего места — все ключи

| Ключ | Тип | Кто ставит | Семантика |
|---|---|---|---|
| `workplace_no` | str | extract | Канонический номер: 6 цифр с ведущими нулями + опц. суффикс (`"000012а"`). Номер — ТОЛЬКО из содержимого карты (строка `…-сонли`); нет строки → `""` → карта в «НЕ РАСПОЗНАНЫ». ⚠️ `int(workplace_no)` сломается на суффиксе |
| `workplace_no_in_doc` | str | extract | Сырой номер из документа без паддинга (для сверки) |
| `source_file` | str | extract | Имя исходного .doc |
| `position` | str | extract | Должность: следующий непустой абзац после якоря «ИШ ЖОЙИ НОМИ»; фолбэк — из имени файла |
| `position_from_filename` | str | extract | Название из имени файла (запасной источник) |
| `position_from_perechen` | str | pipeline | Должность из «Перечня» (в 6_5 приоритетна) |
| `subdivision` | str | extract | Подразделение из таблицы «Таркибий бўлинма», канонизировано через `SUBDIVISION_ALIASES` |
| `job_code` | str | pipeline | Код должности из «Перечня» по номеру РМ; в extract всегда `""` |
| `factors` | dict[str,str] | extract | 15 ключей (см. ниже). Значения: класс `"2.0"`, `"3.1"`… или `"-"` (фактора нет) |
| `substances` | list[dict] | extract | Вредные вещества для 5_1б: `{"name": str, "pct": str}`, в каноничном порядке `SUBSTANCE_ORDER` |
| `ppe_provided` | `"ҳа"/"йўқ"` | extract | Обеспеченность СИЗ из таблицы СИЗ (для 5_1б/6_5/ревью) |
| `benefits` | dict[str,str] | extract | 4 льготы из раздела 4.1 карты, значения `"ҳа"/"йўқ"`: `extra_leave`, `reduced_hours`, `therapeutic_food`, `milk` |
| `privileged_pension` | `"ҳа"/"йўқ"` | extract | Из текста раздела 4.2 («…пенсия таъминоти ҳуқуқи __ҳа__») |
| `injury_risk` | `"1"/"2"` | extract | **Эвристика** «медик→1, иначе→2» (колонка c18 6_5); всегда флаг `injury_risk_heuristic` |
| `employees_count` | str-число | extract | Из строки «Ишловчилар сони» карты; не найдено → `"1"` + флаг |
| `female_count` | str-число или `""` | extract | Из строки «жумладан аёллар» или колонки «Аёллар»; не найдено → `""` (НЕ «0»!) + флаг |
| `injury_risk_class_6_4` | `"1"/"2"/"3"/""` | extract | Класс травмоопасности из п.2.3 карты (пропуск `____N____`); `""` = не извлечено, render_6_4 громко предупредит |
| `ppe_status_6_4` | `'mos'/'mos_emas'/'kutilmagan'` | extract | Статус ЯТҲВ из п.3.3 (наличие абзаца = «Мос») |
| `ppe_not_envisaged_6_4` | bool | extract | «СИЗ не предусмотрены» из таблицы СИЗ п.3.2 (нет ни одного реального наименования). НЕЗАВИСИМ от ppe_status |
| `subdivision_6_4` | str | pipeline | Подразделение из строки-разделителя «Перечня». **Наличие ключа = позиция учтена в 6_4** |
| `employees_count_6_4` | str | pipeline | Численность из колонки «Ишловчилар сони» «Перечня» (6_4 считает ЛЮДЕЙ только отсюда, не из карты) |
| `female_count_6_4` | str | pipeline | Из колонки «Шундан аёллар» «Перечня» |
| `physical_measurements` | dict | extract | Excel-протокол 2: `{noise, vibration_local, vibration_general, infrasound}` → замер `{norma,actual,time,cls}` или `None` |
| `microclimate_measurements` | dict | extract | Excel-протокол 3: `{category_label, temp, air_speed, humidity, heat_radiation}` (активная категория 1.8.x; метка — из 1.7.x) |
| `lighting_measurements` | dict | extract | Excel-протокол 4: `{discharge, natural, combined, artificial, pulsation}` (разряд зрит. работ + КЕО/лк/пульсация 1.11.*) |
| `em_measurements` | dict | extract | Excel-протокол 5: `{"1.4.1"…"1.4.10"}` → замер или `None` (у медиков заполнена обычно только 1.4.10) |
| `flags` | list[str] | оба | См. ниже |

`substances` дополнительно к `name`/`pct` несёт `norma`/`actual`/`cls` (для Excel-
протокола 1); порядок SUBSTANCE_ORDER не изменился (5_1б читает только name/pct).
Замер (`_measurement`) — dict `{norma, actual, time, cls}`: строки, где `norma`
может быть диапазоном `«23,0-31,0»`, `actual` — числом или текстом `«йўқ»` (тепло-
излучение), `«-»`/`None` = замера нет. Раздел карты → поле см. `mapping.*_SECTIONS`
и `05-RENDER.md` (раскладка Excel-протоколов). «а»-суффикс наследует эти поля от
базового (см. `_SUFFIX_COPY_FIELDS`), как остальные факторные данные.

### factors — 15 фиксированных ключей

Подытоги (строка-якорь в таблице факторов, значение из колонки класса):
`chem`, `microclimate`, `lighting`, `severity`, `intensity`, `overall`.
По номеру раздела (max класс среди строк раздела): `biological` (1.2),
`aerosols` (1.1.6), `noise` (1.3.2), `infrasound` (1.3.1), `ultrasound_air` (1.3.3),
`vibration_general` (1.3.6), `vibration_local` (1.3.5), `em_field` (1.4),
`ionizing` (1.12).

## Флаги — полный справочник

Флаги = «поля, требующие проверки оператором»; подсвечивают ячейки экрана ревью янтарным.

| Флаг | Кто ставит | Условие / смысл |
|---|---|---|
| `injury_risk_heuristic` | extract | **Всегда, у каждой записи** — травмоопасность c18 всегда эвристика |
| `overall_missing` | extract | Общий класс условий труда не извлечён (`""` или `"-"`) |
| `employees_count_missing` | extract | Нет строки «Ишловчилар сони» — значение форсировано в `"1"` |
| `female_count_missing` | extract | Нет гендерной разбивки — `female_count == ""` (влияет на 6_4) |
| `position_mismatch` | pipeline | Должность в «Перечне» ≠ должности в карте на одном номере (fold-сравнение) |
| `job_code_missing` | pipeline | Номера нет в «Перечне» (для «а»-суффиксов снимается после наследования кода) |
| `substances_missing` | pipeline | В карте нет раздела веществ — в 5_1б будет строка с пустой ячейкой |
| `copied_from_base` | pipeline | Данные скопированы из базового номера («а»-суффикс с пустыми факторами) |
| `card_missing` | pipeline | Карты нет вовсе — запись синтезирована из базовой (номер был в «Перечне») |

Флагов вида `*_inferred` (упоминаются в старом README) в коде **нет** — единственная
эвристика с флагом это `injury_risk_heuristic`.

## Конвенции значений

- **Да/нет** — только строки `"ҳа"` / `"йўқ"` (узб. кириллица: ҳ=U+04B3, ў=U+045E,
  қ=U+049B). Не bool! Сравнивать по точным строкам, канонизация — `normalize.canon_yesno`.
- **Классы** — строки `"X.Y"` (`"3.1"`), голое `2` нормализуется в `"2.0"`;
  «нет фактора» = `"-"`.
- **Пустые значения различаются**: `""` = «не извлечено/неизвестно», `"-"` = «нет»,
  `"0"` почти не используется (в 6_4 ноль рендерится прочерком). Исключение:
  `employees_count` при промахе = `"1"` (карта описывает занятое место).
- Три параллельных источника должности (`position`, `position_from_filename`,
  `position_from_perechen`) — механизм кросс-валидации; расхождение = `position_mismatch`.
