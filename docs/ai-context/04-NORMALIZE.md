# Нормализация, двуязычие, конвертация, распаковка

## normalize.py — чистые функции без состояния

### Три разных транслитерации — не путать

| Функция | Назначение | Свойства |
|---|---|---|
| `fold(s)` | ТОЛЬКО для сравнения якорей | лосси: lower, кир→лат, апострофы и `ъ/ь` выброшены, всё кроме `[0-9a-z ]` → пробел. Гарантия: `fold('баҳолаш') == fold('baholash')` |
| `to_latin(s)` | Читаемый вывод (язык документа `lat`) | сохраняет регистр и апострофы: `ў→o'`, `ғ→g'`, `ъ→'`; ye-правило для `е` |
| `to_cyrillic(s)` | Обратная (лат→кир) | диграфы длинным совпадением: `o'/o`/oʻ→ў`, `sh→ш`, `ch→ч`, `yo→ё`, `ts→ц`; спец-правило `yo'` = й+ў (не ё) |

Детали `fold`: таблица `_CYR2LAT` (нижний регистр, `ў→o`, `ғ→g`, `х→x`, `ҳ→h` —
поэтому `fold("ҳа")=="ha"`, но `fold("ха")=="xa"` — НЕ равны, оба варианта есть в
YES-токенах); апострофы `` `'’ʻ‘ʼ´ʹ `` удаляются; `щ` в fold-таблице отсутствует —
как и любой неотображённый символ, превращается в пробел (разрыв слова);
в `to_latin` буква есть (`щ→sh`).

Детали `to_latin`: ye-правило — `е → "ye"` в начале слова или после гласной, иначе
`"e"` (`Ели → Yeli`, `мен → men`); регистр переносится по первой букве (`Ч → Ch`).

Детали `to_cyrillic`: `c→к`, `w→в` (заимствования); одиночный апостроф между буквами
молча выбрасывается. ⚠️ Хрупкости: (1) `yo` в самом конце строки даёт `йо`, а не `ё`
(пустой срез `"" in _APOSTROPHES` == True); (2) типографские апострофы `’ ‘ ʼ ´ ʹ`
не распознаются в диграфах `o'/g'` — `o’` станет `о`, не `ў`.

### Значения

- `normalize_spaces` — NBSP→пробел, удаление невидимых (`U+200E/200F/200B/2060/FEFF`),
  схлопывание пробелов. Первый шаг почти всех функций.
- `is_empty` — пусто, если после нормализации ∈ `{"", "-", "—", "–", "−", "нет", "yoq"}`.
  ⚠️ `-` считается пустым — в трёх местах extract это специально обходится
  (реквизиты, колонка класса, колонка «Аёллар»).
- `canon_yesno` → `"ҳа"/"йўқ"`: точные токены (YES: `ҳа, ха, бор, +, да, yes, ha,
  bor, xa`; NO: `йўқ, йуқ, йук, yo'q, yo`q, yoq, нет, no, yuq` + недостижимый
  `yo'q.` — точка и так срезается `rstrip(".")`), затем префиксы для
  составных ячеек («ҳа, 3-илова…»), причём «нет»-префиксы проверяются РАНЬШЕ.
  **Пусто и любой нераспознанный текст → `"йўқ"`** (консервативно).
- `normalize_class` → `"X.Y"`: `3. 3`/`3,3` → `"3.3"`; голое `2` → `"2.0"`; мусор
  проходит насквозь. `class_rank` (`3.3`→33, невалид→-1), `max_class` — худший класс.
- `normalize_number` — склейка пробелов между цифрами (`8 0`→`80`), пробелы вокруг
  `.`/`,` убираются (разделитель не меняется).
- `clean_substance_name` — срез единиц измерения с конца (`_UNIT_RE`: `мг/м3`,
  `мг/м³`, `мг/куб.м`, `мкг/м3`, `г/м3` + латинские `mg/m3` и т.п.).
- `_to_int` — первое целое из текста, `-`/пусто → 0 (используется рендером 6_4).

## convert.py — LibreOffice headless

- `find_soffice()`: env `SOFFICE_BIN` (без валидации!) → PATH (`soffice`,
  `libreoffice`) → `/Applications/LibreOffice.app/Contents/MacOS/soffice` →
  `/usr/bin/soffice` → `/usr/bin/libreoffice`. Ничего → `ConversionError`.
- `convert_to_docx(src, out_dir, timeout=180)` / `convert_to_doc(...)` → `_convert`:
  - каждый вызов — **уникальный профиль** `-env:UserInstallation=file:///tmp/lo_<uuid>`
    (soffice не переносит параллельные запуски с общим профилем), профиль удаляется
    в finally;
  - вызовы сериализованы модульным `threading.Lock` — **concurrency=1 на процесс**
    (между процессами защита — уникальные профили);
  - флаги: `--headless --norestore --nolockcheck --convert-to <target> --outdir …`;
  - успех = `returncode == 0` И существует `out_dir/<stem>.<target>` (soffice умеет
    возвращать 0 при провале);
  - ⚠️ `subprocess.TimeoutExpired` НЕ оборачивается в ConversionError; одинаковые
    stem-имена в одном out_dir перезаписывают друг друга; залежавшийся старый файл
    может замаскировать провал.
- Причина всей этой осторожности: LibreOffice спайкует до **~1.8 ГБ RAM** на
  конвертации .doc — отсюда и `--concurrency=1` у Celery-воркера, и требование
  swap 2 ГБ на проде (см. `08-DEPLOY.md`).

## unpack.py — распаковка zip

- Проблема: кириллические имена внутри zip часто хранятся **без флага UTF-8** —
  zipfile декодирует их как cp437 и получается мусор. `_decode_name`:
  1. флаг UTF-8 (0x800) установлен → имя уже верное;
  2. пробуем `name.encode("cp437")` — если UnicodeEncodeError, строка уже содержит
     кириллицу и корректна;
  3. иначе декодируем сырые байты: `utf-8` → `cp866` → `cp1251` (cp866 декодирует
     любые байты, так что cp1251 — мёртвая страховка).
- Фильтры мусора: каталоги, `__MACOSX` (подстрокой по всему пути), префиксы
  `~$ ~wrl ~wrd .~ ._`, суффиксы `.tmp .ds_store`.
- Zip-slip: компоненты `..`, `.`, `""` вырезаются из пути (⚠️ POSIX-абсолютное имя
  `/etc/x` теоретически не отфильтровано — компонент `/`).
- Возвращает `list[UnpackedFile]`: `arc_name` (декодированное имя с подпапками),
  `abs_path`, свойства `basename` (бэкслэши нормализованы), `suffix` (lower).
- Файл читается в память целиком (`src.read()`); коллизии декодированных имён не
  детектятся (последний перезапишет).
