# Авто-аттестация — полный обзор проекта

> Этот пакет документации (`docs/ai-context/*.md`) написан для загрузки в контекст
> AI-ассистента: он описывает проект целиком — домен, архитектуру, форматы данных,
> все неочевидные решения и хрупкие места. Файлы пронумерованы, читать по порядку.

## Что это

Веб-приложение (Django), которое автоматически заполняет три фиксированных
Word-документа (**5_1б**, **6_5**, **6_4**) данными, извлечёнными из zip-архива
аттестации рабочих мест. Заменяет ручной копи-паст: **загрузил архив → проверил
данные на экране ревью → скачал готовые документы**.

Предметная область — **аттестация рабочих мест** (оценка условий труда) для
организаций Узбекистана. Документы на узбекской кириллице и узбекской латинице.

## Глоссарий домена

| Термин | Значение |
|---|---|
| **Карта (рабочего места)** | .doc-файл на одно рабочее место: условия труда, факторы, СИЗ, льготы. Все карты сгенерированы из одного гос-шаблона |
| **РМ** | Рабочее место. Номер вида `000012`, бывают «а»-суффиксные варианты `000012а` (доп. место/смена того же типа) |
| **«Перечень»** | Отдельный .doc — список всех рабочих мест с кодами должностей и численностью, сгруппированный по подразделениям |
| **5_1б** | Выходной документ: вредные вещества по рабочим местам (3 колонки) |
| **6_5** | Выходной документ: большая сводная таблица, 25 логических колонок на каждое РМ |
| **6_4** | Выходной документ: «йиғма қайднома» — итоги аттестации ПО ПОДРАЗДЕЛЕНИЯМ (не по РМ) |
| **ЯТҲВ** | СИЗ (средства индивидуальной защиты), узб. аббревиатура |
| **ҳа / йўқ** | «да / нет» на узбекской кириллице — канонические значения всех да/нет-полей в датасете |
| **Эталон** | Заполненный клиентом «правильный» документ («…қилиниши керак») — база для самопроверки |
| **СТИР / ИФУТ / МХБТ** | Коды организации (аналоги ИНН / ОКЭД / СОАТО) |
| **Бухоро** | Первый клиент, карты на узбекской **кириллице** (архив `Бухоро болалар стоматологияси.zip`) |
| **RIZOYEV** | Второй клиент, карты на узбекской **латинице** (архив `KARTA.zip`, эталоны — отдельные файлы `5_1б_правилный.doc`, `6_5-правилный.doc`) |

## Ключевые архитектурные принципы

1. **Единый промежуточный JSON.** Никогда не «карта → шаблон» напрямую. Сначала всё
   извлекается в один чистый датасет (одна запись на РМ + блок реквизитов компании),
   затем ВСЕ шаблоны заполняются из него. Новый шаблон = новый рендерер (так добавлен 6_4).
2. **Якоря вместо координат.** Данные ищутся по подстрокам текста («якорям» из
   `attestation/services/mapping.py`), а не по номерам строк/ячеек — карты
   генерировались из одного гос-шаблона, но вёрстка плавает.
3. **Двуязычие через `fold()`.** И текст документа, и якоря транслитерируются к общей
   форме (`normalize.fold`: узб. кириллица → латиница, апострофы и регистр отбрасываются).
   Один кириллический набор якорей работает и для карт Бухоро, и для карт RIZOYEV.
4. **Верим файлам, а не именам и не ТЗ.** Номер РМ и должность — из содержимого карты,
   а не из имени файла; при расхождении ТЗ и реальных эталонов — верим эталонам
   (см. `09-GOTCHAS.md`).
5. **Громкие предупреждения вместо тихих потерь.** Нераспознанная карта, отсутствующий
   класс, несматченное подразделение — всегда warning, который видит оператор.
6. **Selfcheck — регрессионный гейт.** Перед любыми изменениями парсера/рендера:
   `python manage.py selfcheck "Бухоро болалар стоматологияси.zip"` должен давать
   **5_1б — 41/41 рабочих мест, 6_5 — 1012/1032 ячеек** («Самопроверка пройдена»).
   ⚠️ Именно 1012, НЕ 1032: 20 ячеек c22 «Сут» у медиков всегда уходят в notes
   (эталон Бухоро вручную ставит молоко=ҳа вопреки картам; мы выводим значение
   карты — сознательное решение 2026-06-24, «верим картам»). notes гейт НЕ
   проваливают (`ok = total>0 and not mismatches`); mismatches тут = 0.
   Excel-протоколы (5 xlsx) сверяются отдельно:
   `python manage.py selfcheck <sud.zip> --excel-dir excel_templates` →
   **3461 совпавшая ячейка, 0 mismatch** (файл 1 — 1572, 2 — 263, 3 — 832,
   4 — 629, 5 — 165; расхождения = опечатки эталона Суд → notes, см. `07-SELFCHECK.md`).

## Технологический стек

- Python 3.12, **Django 5.1.4**
- **LibreOffice (headless)** — конвертация `.doc → .docx` (python-docx не читает старый .doc)
- **python-docx 1.2.0** — чтение таблиц и заполнение шаблонов
- **Celery 5.4 + Redis** — фоновая обработка (архив ~100 МБ, десятки конвертаций);
  локально вместо Celery — фоновый поток (`ATTESTATION_TASK_RUNNER=thread`)
- **SQLite** в dev, **PostgreSQL 16** в проде (переключение по наличию env `POSTGRES_DB`)
- Фронт: Django templates + **Tailwind CSS (CDN)** + **HTMX** (поллинг прогресса,
  инлайн-сохранение ячеек) + **Alpine.js** (drag-and-drop)

## Быстрый старт

```bash
# Docker (web + worker + redis, LibreOffice в образе):
docker compose up --build
docker compose exec web python manage.py createsuperuser

# Локально без Docker (нужен LibreOffice; DEBUG=True → раннер thread, Redis не нужен):
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python manage.py migrate && python manage.py runserver
```

Полезные команды:

```bash
# Извлечение → JSON без БД/UI (отладка якорей):
python manage.py extract_demo "Бухоро болалар стоматологияси.zip" --out media/demo.json

# Самопроверка (регрессионный гейт, ненулевой код при расхождении):
python manage.py selfcheck "Бухоро болалар стоматологияси.zip"
python manage.py selfcheck KARTA.zip --bilingual --ref-5-1b 5_1б_правилный.doc --ref-6-5 6_5-правилный.doc
```

## Структура репозитория

```
config/                  # Django-проект: settings, urls, celery, wsgi/asgi
core/                    # пустое общее приложение (задел)
attestation/
  models.py              # Batch (JSON-поля датасета), SourceFile
  jobs.py                # оркестрация этапов + переключатель thread/celery
  tasks.py               # Celery-обёртки (2 задачи)
  views.py / urls.py     # экраны: dashboard, upload, detail, HTMX-эндпоинты
  admin.py               # админка Batch/SourceFile
  services/              # ЧИСТАЯ логика конвейера, БЕЗ зависимости от Django:
    unpack.py            #   распаковка zip (кир. имена, фильтр мусора)
    convert.py           #   обёртка LibreOffice .doc→.docx
    normalize.py         #   fold/транслитерация, числа, ҳа/йўқ, классы
    mapping.py           #   ДЕКЛАРАТИВНЫЙ конфиг якорей (правьте тут)
    extract.py           #   движок извлечения карт и «Перечня»
    pipeline.py          #   оркестрация unpack→convert→extract→датасет
    render.py            #   заполнение шаблонов 5_1б, 6_5, 6_4
    selfcheck.py         #   сравнение с эталонами
  assets/                # пустые шаблоны: template_5_1b.docx, template_6_5.docx, template_6_4.docx
  management/commands/   # extract_demo, selfcheck
templates/               # base + login + dashboard/upload/detail + HTMX-фрагменты
nginx/, scripts/, docs/  # прод-инфраструктура
Dockerfile, docker-compose.yml, docker-compose.prod.yml
```

В корне репозитория лежат (не в git): тестовые фикстуры `Бухоро болалар
стоматологияси.zip`, `5_1б_правилный.doc`, `6_5-правилный.doc` — без них selfcheck
не воспроизводится на другой машине (`.gitignore` исключает `*.zip`, `*.doc`).

## Объём MVP

**Входит:** загрузка → фоновая обработка → извлечение → ревью с инлайн-правкой и
подсветкой → генерация трёх документов (с выбором языка вывода Кириллица/Lotin) →
скачивание; самопроверка против эталонов; docker-compose; прод-деплой с CI/CD.

**Не входит:** мультиарендность и роли (любой залогиненный видит все батчи); OCR
сканов; произвольные новые шаблоны; LLM-fallback (есть только флаг
`ATTESTATION_LLM_FALLBACK`, по умолчанию выключен, реализация-хук).
