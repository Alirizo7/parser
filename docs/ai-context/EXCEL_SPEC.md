# Спецификация 5 Excel-протоколов (приложение к задаче)

> Данные извлечены из реальных шаблонов (5 xlsx) и сверены с реальными картами
> клиента «Суд-экспертиза Қарши» (латиница, 52 РМ). Совпадение карта→Excel — точное.

## Общий принцип (единый для всех 5 файлов)

Каждый файл = один лист `complete` (лишний `Лист1` в файлах 4 и 5 — мусор чужого
клиента, при построении шаблона-ассета удалить). Структура листа:

```
[шапка: строки 1..~25]  — статичная, приборы/НД/лаборатория зашиты в шаблон,
                          МЕНЯЕТСЯ только имя/адрес заказчика (company_data)
[строка(и) заголовков колонок таблицы]
[тело: чередование <заголовок-подразделения на всю ширину A:K> + <блоки РМ>]
```

Тело генерируется заново из датасета (как в docx-рендерах): клонирование
прото-строк, вписывание значений, транслитерация всего документа по `output_lang`.

Псевдозамеры E/F/G (колонки «1-фаолият/2-фаолият/3-фаолият») — это **формулы от
факта H (или I)**, уже прошитые в прото-строке. При клонировании строки формула
сама сошлётся на свою строку (относительные ссылки). Вписывать надо только H (факт).

## Раскладка колонок

### Файл 1 — Вредные вещества (12 колонок A..L, СДВИНУТ на 1 vs остальные)
Одна строка Excel = одно вещество. РМ = группа строк, номер только в первой.
| Кол | Заголовок | Источник (из карты, раздел 1.1.*) |
|-----|-----------|-----------------------------------|
| A | Ish o`rni raqami | workplace_no (первая строка группы) |
| B | O`lchov nomi, ish nomi | position (первая строка группы) |
| C | Identifikatsiya (вещество) | имя вещества |
| D | Xavfli sinf (класс опасн. III/IV, суффикс O/Ф) | **НЕ из карты — оставить пусто/«-»**, оператор впишет в ревью |
| E | PDK (ПДК/норма) | норма из карты (кол. «гигиеник меъёр») |
| F | 1-faoliyat/haqiqiy | ФОРМУЛА `=I{r}+0.1` |
| G | 2-faoliyat/haqiqiy | ФОРМУЛА `=I{r}+0.2` |
| H | 3-faoliyat/haqiqiy | ФОРМУЛА `=I{r}-0.3` |
| I | O`rtacha haqiqiy | ФАКТ из карты (кол. «ҳақиқий даражаси») |
| J | Ta`sir muddati (soat/%) | время из карты (кол. «давомийлиги») |
| K | Mehnat sharoitlari klassi | класс строки вещества из карты |
| L | Yakuniy klass (итог по РМ) | «зарарли моддаларни умумий баҳолаш» (только 1-я строка РМ) |

### Файлы 2–5 — общий шаблон (11 колонок A..K)
Одно РМ = фиксированный набор под-строк (см. ниже). Номер РМ и должность —
только в первой под-строке блока (вертикальный merge A и B на весь блок).
| Кол | Заголовок | Источник |
|-----|-----------|----------|
| A | Ish o`rni T/r | workplace_no |
| B | ish joyi nomi | position |
| C | показатель (метка под-строки) | статичная метка (из шаблона) |
| D | PDU (норма) | норма из карты для этой под-строки |
| E | 1-faoliyat | ФОРМУЛА `=H{r}+шаг` |
| F | 2-faoliyat | ФОРМУЛА `=H{r}+2·шаг` |
| G | 3-faoliyat | ФОРМУЛА `=H{r}-3·шаг` |
| H | O`rtacha haqiqiy | ФАКТ из карты |
| I | Ta`sir muddati (soat/%) | время из карты |
| J | Mehnat sharoitlari klassi | класс из карты |
| K | Yakuniy klass | итог по фактору (merge на весь блок, 1-я под-строка) |

Шаг формул по файлам: файл 2 (физич) — целые `±1/+2/−3`; файлы 3/4/5 — `±0.1/0.2/0.3`
для микроклимата, `±0.01/0.02/0.03` для света и ЭМИ.

## Под-строки блока РМ (порядок фиксированный, из шаблона)

### Файл 2 — Физические факторы (4 под-строки)
0. `Shovqin, dBA`          ← карта 1.3.2 (Shovqin tovush darajasi…dBA)
1. `Tebranish: mahalliy, dB` ← карта 1.3.5.* (локальная вибрация)
2. `Umumiy, dB`             ← карта 1.3.6.* (общая вибрация)
3. `Infratovush, dB`        ← карта 1.3.1 (инфразвук)

*(В этом клиенте у всех РМ заполнена только строка Shovqin; остальные пустые —
норма/факт/класс из карты «−», формулы всё равно есть в прото-строке.)*

### Файл 3 — Микроклимат (9 под-строк)
0. `Ishlarning umumiy energiya sarfi Vt/m2` — статич. метка, только K=итог
1. `Ishlar toifasi.` — D = категория работ (Iб/IIб…) из карты (какой раздел 1.8.x
   активен: карта содержит несколько строк 1.8.1…1.8.5 для разных категорий; берётся
   та, у которой заполнена норма/факт). БЕЗ E/F/G/H.
2. `Harorat, оС` ← карта 1.8.x (температура, строка активной категории)
3. `Havo tezligi, m/s` ← карта 1.8.7
4. `Nisbiy namlik, %` ← карта 1.8.8
5. `Issiqlik nurlanish Vt/m2` — D=100, E/F/G/H = «yo'q» (текст, не число)
6. `Harorat, оС (ochiq hudud)` — пусто
7. `WBGT ko`rsatkichi` — пусто
8. `Issiqlikning o`rtacha yuki TNS` — пусто

### Файл 4 — Освещённость (5 под-строк)
0. `Vizual ishlarni bajarish` — D = разряд зрит. работ (III/IV) из карты 1.11
   (строка «Vizual ish oqimi III/IV»); только K=итог, без E/F/G/H
1. `Tabiiy yorug`lik (КЕО %)` ← карта 1.11.1 (естеств. освещённость; D часто «-»)
2. `Birlashtirilgan yorug`lik (КЕО %)` ← карта 1.11.2 (смешанная)
3. `Sun`iy yorug`lik…yoritilishi (Е, lx)` ← карта 1.11.3 (искусственная, лк)
4. `Pulsatsiya YoP %` ← карта 1.11.6 (пульсация; часто пусто)

### Файл 5 — Магнитные поля / ЭМИ (16 под-строк)
0. `Doimiy magnit maydon, kA/m` ← карта 1.4.1
1. `Elektrostatik maydon, kV/m` ← карта 1.4.2
2. `Quvvat chastotasi elektr maydonlari (50 Гц), kV/m` ← 1.4.3
3. `Quvvat chastotasi magnit maydonlari (50 Гц), kA/m` ← 1.4.4
4-5. `Radiochastotali…: 0.01-0.03 МГц, V/m` ← 1.4.5 (метка в 2 строки: C4 общий текст, C5 диапазон)
6-7. `0.03-3.0 МГц` ← 1.4.6
8-9. `3.0-30.0 МГц` ← 1.4.7
10-11. `30.0-300.0 МГц` ← 1.4.8
12-13. `300.0 МГц-300.0 ГГц, mkVt/kv.sm` ← 1.4.9
14-15. `VDT va PEVM…, V/m` ← карта 1.4.10 (D=2.5) — **единственная реально

*(У этого клиента 1.4.1–1.4.9 везде «−», заполнена только 1.4.10. Но логика
рендера должна уметь вписать любую из 1.4.* если в карте есть замер.)*

## Группировка по подразделениям

Файлы **1, 2, 3, 4** — С группировкой: перед первым РМ группы строка-заголовок
merged A:K с названием подразделения. Порядок и названия групп = как в docx 6_5
(по смежности `subdivision`) и как в «Перечне». Прото-строку заголовка группы
клонировать из существующей merged-строки шаблона.

Файл **5 (магнитные) — БЕЗ группировки**: все РМ идут подряд, без строк-заголовков
подразделений. (Проверено на образце: 33 РМ подряд, ни одной merged A:K строки.)

## Логика активной строки микроклимата (файл 3) — тонкий момент

В карте раздел 1.8 содержит 5 строк температур для разных категорий работ
(1.8.1=Iа, 1.8.2=Iб, 1.8.3=IIа, 1.8.4=IIб, 1.8.5=III). У РМ активна ОДНА — та, где
колонка «факт» непустая (у остальных «−»). Пример: карта 052 → активна 1.8.4 (IIб),
факт температуры 25,0. Рендер:
- под-строка `Ishlar toifasi.` D = категория активной строки (напр. «IIb – 145(130-160)»
  — можно взять из текста метки активной строки карты, либо из отдельной строки
  «Ishlar toifasi» карты, проверить где надёжнее);
- под-строка `Harorat, оС` факт = факт активной строки 1.8.x;
- `Havo tezligi` ← 1.8.7, `Nisbiy namlik` ← 1.8.8 (эти всегда фиксированные номера).

## Шапка (строки 1..~25) — что переменное

Зашито в шаблон (одинаково для всех клиентов, НЕ трогать):
лаборатория, адрес лаборатории, аккредитация, приборы (АНТ-3М/ВШВ-003/Метеоскоп/
Аргус/ПЗ-41), НД (ГОСТы), даты поверки, «harorat 23.0», subpudratchi «yo'q».

Переменное (из company_data):
- `Buyurtmachi nomi:` / `Наименование заказчика:` → company_data.name
- `Buyurtmachi manzili:` / `Адрес заказчика:` → company_data.address

(В существующих шаблонах там зашит старый клиент — при построении ассета можно
оставить как placeholder-текст; рендер перезапишет строку после метки «заказчик».)

## Язык вывода

Тот же переключатель `output_lang` (cyr/lat), что у docx. Транслитерация всего
листа через существующие `to_latin`/`to_cyrillic`. ВАЖНО: заголовки колонок в
шаблонах СЕЙЧАС смешанные — в latin-шаблонах Суд они на латинице, в cyr-шпорах на
кириллице. Как в docx-рендере: держать прото-шаблон на ОДНОМ письме (кириллице) и
транслитерировать целиком по output_lang. Значения из карт тоже фолдить/выводить
через ту же машинерию.
